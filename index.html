How it runs

One command,
six moving parts.

forgex investigate TARGET --profile NAME quietly wires together every piece below. This is the actual order of operations, not a marketing sequence.

01

Evidence added

Source is hashed and logged to the chain-of-custody record. Nothing is copied unless you ask.

02

Profile selects modules

A profile (YAML) picks which of the 9 artifact modules and which rules apply.

03

Modules parse artifacts

Disk, filesystem, Windows, network, malware, browser… each domain runs its own parser.

04

Rules produce findings

Detection rules score what the modules surfaced — severity, confidence, evidence refs.

05

Correlate & timeline

Findings are linked into a graph and merged into a single reconstructed timeline.

06

Report rendered

JSON, Markdown, HTML, CSV or PDF — one call, same underlying result.

Artifact modules

Nine domains, one interface.

Every profile is built from these. Call them directly too — forgex windows evtx, forgex network ja3, and so on.

disk

DD / E01 / VHD / VHDX / QCOW2 / VMDK image analysis & partition parsing.

filesystem

Tree walks, deleted-file recovery, ADS, slack space, MAC-time timelines.

windows

Registry hives, EVTX, $MFT, $UsnJrnl, Prefetch, LNK — all native, no deps.

linux

System & user artifacts — logs, cron, shell history, service units.

macos

LaunchAgents/Daemons, plists, unified logs, Spotlight metadata.

browser

History, downloads, cookies and cache artifacts across major browsers.

network

pcap / pcapng decoding, capture summaries, TLS ClientHello / JA3.

malware

PE/ELF/Mach-O header analysis, entropy, built-in YARA-lite engine.

ioc

Extracts IPs, domains, hashes and emails out of any text or log file.

Command reference

Every command speaks JSON.

Add --json to anything below for machine-readable output — scriptable by design.

forgex evidence

Catalog, chain of custody, and hash verification against the current case.

$ forgex evidence add ./evidence.E01 --copy --notes "Workstation-07, seized 2026-07-10" $ forgex evidence list --json $ forgex evidence verify a1b2c3d4 # exits 2 on hash mismatch — CI-friendly $ forgex evidence export a1b2c3d4 ./exports/evidence.E01

forgex investigate

Runs a profile against a target and produces findings.

$ forgex investigate ./case_dir --profile quick $ forgex investigate ./case_dir --profile ransomware --format html -o report.html $ forgex investigate ./sample_dir --profile malware --json | jq '.findings'

forgex report

Render a finished investigation as a shareable file.

$ forgex report formats --json # json · markdown · html · csv · pdf (pdf needs the "full" extra)

forgex disk

Disk image analysis — DD / E01 / VHD / VHDX / QCOW2 / VMDK.

$ forgex disk info ./evidence.E01 $ forgex disk partitions ./evidence.E01 --json $ forgex disk mount ./evidence.vhd ./mnt --partition 1

forgex fs

Filesystem operations against a live directory or mounted volume.

$ forgex fs tree ./extracted_evidence --max-depth 3 $ forgex fs search ./extracted_evidence "*.ps1" --content --json $ forgex fs timeline ./extracted_evidence --out timeline.json

forgex windows

Native parsing — no native deps required — for registry, EVTX, MFT, USN, Prefetch, LNK.

$ forgex windows registry ./SYSTEM "ControlSet001\Services" $ forgex windows evtx ./Security.evtx --max-records 500 --json $ forgex windows mft ./$MFT --json > mft_records.json

forgex network

pcap / pcapng parsing, plus TLS ClientHello / JA3 fingerprinting.

$ forgex network summarize ./capture.pcapng $ forgex network packets ./capture.pcap --max-packets 200 --json $ forgex network ja3 ./client_hello.bin

forgex util

Standalone one-off checks that don't need a case — hash, entropy, strings, and more.

$ forgex util entropy ./sample.bin # high entropy → possible packing/encryption $ forgex util strings ./sample.bin --min-length 6 $ forgex util ioc ./browser_history_dump.txt --json $ forgex util doctor # check optional-dependency status

forgex plugin

Confirm what your plugin directory registered before it's mid-investigation.

$ forgex plugin list --json { "loaded_plugins": [...], "commands": [...], "rules": [...], "profiles": [...], "report_formats": [...] }
Install & setup

Runs anywhere.
Requires Python 3.10+.

Use the guided installer, or install directly with pip. Either way, forgex lands on your PATH and works from any directory — not just the repo you cloned.

Guided install

> python install.py

Walks you through OS detection, installs python-magic-bin in place of python-magic, and flags that PDF reports need the GTK3 runtime installed separately.

Manual

> pip install "forgex[full]" > forgex --help

Core Forgex has zero native dependencies — the [full] extra is optional, for deeper artifact parsing.

Guided install

$ python install.py

Auto-detects your distro family from /etc/os-release (Ubuntu/Debian, Fedora/RHEL, Arch, openSUSE, Alpine — and everything based on them) and offers to install native libraries via the right package manager before you confirm anything with sudo.

Manual

$ pip install "forgex[full]" $ forgex --help # externally-managed environment? $ pip install "forgex[full]" --break-system-packages

Guided install

$ python install.py

Offers to install native libraries (libmagic, pango, gdk-pixbuf, libffi) via Homebrew if it's on your PATH.

Manual

$ brew install libmagic $ pip install "forgex[full]" $ forgex --help
  • Check your environment. forgex util doctor — Python version, platform, optional-dependency status.
  • Add evidence. forgex evidence add ./suspicious_dir --notes "Initial triage" hashes and catalogs it, without touching the source.
  • Investigate. forgex investigate ./suspicious_dir --profile quick --json
  • Report. Add --format html -o report.html for a file you can hand off.
